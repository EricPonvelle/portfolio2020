What is this document?

API documentation is never fun. Writers must understand how all these compoents and endpoints and logic work together, and developers must make sense of this fog of ideas and create a workable solution.

My rationale with this page is to create a more visually appealing API explanation that is still technically accurate and encompassing. This demo is a bit of an overkill for any practical implementations, but pieces of this idea could be introduced to make API documentation more engaging and informative. An audience that feels engaged and interactive is one that will be more receptive to learning.
